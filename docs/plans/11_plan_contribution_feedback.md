Canaliser issues/PR, proposer des profils, enrichir les standards, et intégrer le feedback sans casser la sécurité.

## Labels recommandés
- `type:process` + `ai:docs` (ou `ai:plan` si changements structure)

- [ ] Feature

- [ ] Ouvrir issue `type:compliance`
